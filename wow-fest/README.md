# WOW! Ивент-Фест — лендинг

Flask-приложение для лендинга фестиваля. Структура:

```
app.py                  # роуты: /, /register, /api/seats
sheets_integration.py   # отправка заявок в Google Sheets (webhook или gspread)
apps_script.gs           # код для Google Apps Script (способ 1, см. ниже)
.env.example             # шаблон переменных окружения
templates/               base.html, index.html
static/css/style.css     # стили (палитра и вёрстка по макету)
static/js/main.js        # UTM, форма, счётчик мест, поп-ап, чат-панель
data/                    # registrations.db (sqlite) + seats.json — создаются автоматически
```

## Запуск локально
```bash
pip install -r requirements.txt
python app.py
# открыть http://127.0.0.1:5000
```

## Google Sheets — настройка (способ 1, рекомендуется)
Без Cloud Console и service account, 10 минут:

1. Откройте свою Google Таблицу (создайте новую, если нет).
2. **Расширения → Apps Script**.
3. Удалите весь код в редакторе, вставьте содержимое файла `apps_script.gs` целиком.
4. Сохраните (значок диска или Ctrl+S).
5. **Развернуть → Новое развертывание**:
   - Тип: **Веб-приложение**
   - Кто имеет доступ: **Все**
6. Нажмите "Развернуть", разрешите доступ от своего Google-аккаунта (появится
   предупреждение "Google не проверял это приложение" — это норма для своего
   скрипта: "Дополнительно" → "Перейти на страницу (название проекта)").
7. Скопируйте **URL веб-приложения** (вид `https://script.google.com/macros/s/XXXX/exec`).
8. Локально: `export WOWFEST_SHEETS_WEBHOOK_URL="<этот URL>"` перед запуском
   `python app.py`, либо создайте `.env` из `.env.example` и заполните — Docker
   подхватит автоматически.
9. Проверка: откройте URL веб-приложения в браузере — должно вернуть
   `{"ok":true,"message":"WOW Fest webhook работает"}`. Затем отправьте тестовую
   заявку через форму на сайте — в таблице должна появиться новая строка с
   заголовками (добавляются автоматически при первой записи).

⚠️ Если потом измените код в `apps_script.gs` — нужно создать новое развертывание
(или "Управление развертываниями" → редактировать → новая версия), иначе Google
будет продолжать использовать старую версию скрипта.

### Способ 2 (service account + gspread) — если нужен именно он

## Google Analytics 4 — создание с нуля

1. Перейдите на **analytics.google.com**, войдите в Google-аккаунт.
2. **Администратор** (значок шестерёнки слева внизу) → в колонке "Аккаунт"
   нажмите **Создать аккаунт**.
   - Название аккаунта: например, "WOW Ивент-Фест"
   - Настройки обмена данными можно оставить по умолчанию → Далее
3. **Создать ресурс (Property)**:
   - Название: "WOW Fest Landing"
   - Часовой пояс: Россия, валюта: рубль
4. Заполните базовую информацию о бизнесе (отрасль, размер) → Создать,
   примите условия использования.
5. На шаге **"Настройка сбора данных"** выберите платформу **Web**.
   - URL сайта: ваш домен (или временно `http://localhost:5000`, потом
     поменяете на реальный после деплоя)
   - Название потока: "WOW Fest"
6. После создания потока вы увидите **Measurement ID** вида `G-XXXXXXXXXX`
   — это и есть нужный ID.
7. Впишите его в `.env`:
   ```
   WOWFEST_GA4_ID=G-XXXXXXXXXX
   ```
8. Перезапустите контейнер (`docker compose up --build`) и откройте сайт.
9. **Проверка:** в GA4 откройте **Отчеты → В реальном времени** — должен
   появиться один активный пользователь (вы), пока страница открыта в браузере.

## Facebook (Meta) Pixel — создание с нуля

1. Перейдите на **business.facebook.com/events_manager** (нужен Facebook-аккаунт;
   если рекламного кабинета ещё нет, система предложит создать Business-аккаунт
   по ходу — соглашайтесь на дефолтные настройки).
2. **Источники данных → Подключить источники данных → Интернет** → выберите
   **Meta Pixel** → Подключить.
3. Укажите название пиксела, например "WOW Fest Pixel", и URL сайта.
4. На шаге выбора способа установки выберите **"Установить код вручную"**
   (другие способы — через Tag Manager/партнёров — здесь не нужны).
5. Facebook покажет код с **Pixel ID** — числом вида `1234567890123456`.
   Скопируйте только это число.
6. Впишите в `.env`:
   ```
   WOWFEST_FB_PIXEL_ID=1234567890123456
   ```
7. Перезапустите контейнер.
8. **Проверка:** установите в браузер расширение **Meta Pixel Helper**
   (есть в Chrome Web Store) — откройте сайт, иконка расширения должна
   показать "1 Pixel found" и событие `PageView`. Событие `Lead` появится
   там же после отправки формы регистрации.

## Что уже трекается из коробки
- `PageView` — при каждой загрузке страницы (и GA4, и FB Pixel)
- `generate_lead` (GA4) / `Lead` (FB Pixel) — при успешной отправке любой из
  двух форм (основной регистрации и мини-формы в поп-апе), см. `static/js/main.js`

## Деплой (Render.com, бесплатный план)
1. Запушить проект в GitHub-репозиторий.
2. На render.com → New → Web Service → подключить репозиторий.
3. Build command: `pip install -r requirements.txt`
4. Start command: `gunicorn app:app`
5. Добавить переменную окружения `WOWFEST_SHEETS_WEBHOOK_URL` в Render → Environment.
6. ⚠️ SQLite на бесплатном плане Render не персистентный между деплоями — для надёжности
   стоит держать Google Sheets как основной источник данных, а sqlite — как лог.

## Производительность (требование ТЗ — загрузка до 3 сек)

- Изображения должны быть в WebP и со сжатием перед добавлением в `static/img/`.
- Добавить `loading="lazy"` на все `<img>` ниже первого экрана.
